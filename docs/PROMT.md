АРХИТЕКТУРА И ПРИНЦИПЫ
Основные принципы:
Модульность — каждый модуль отвечает за одну задачу
Асинхронность — используем async/await везде
Обработка ошибок — try/catch с понятными сообщениями
Кэширование по умолчанию — сначала проверяем кэш, потом качаем
Минимальные зависимости — не тащим лишние npm пакеты
IPC для управления — плеер работает в фоне с socket для команд
Поток данных:
1. Пользователь вводит команду → CLI парсит аргументы
2. Если play <query>:
├─ Проверяем кэш
├─ Если нет → ищем через yt-dlp → получаем URL
├─ Запускаем mpv в фоновом режиме с IPC socket
├─ Сохраняем PID и статус в state file
└─ Сохраняем в историю
3. Если play --stop/--pause/--resume:
└─ Отправляем команду через IPC socket в работающий mpv
4. Если трек закончился:
└─ Автозапуск следующего из очереди ИЛИ выход с уведомлением

ОТВЕТЫ НА КЛЮЧЕВЫЕ ВОПРОСЫ
Вопрос 1: Будет ли терминал занят во время воспроизведения?
Ответ: НЕТ, терминал не будет заблокирован.
Реализация:
mpv запускается в фоновом режиме (detached process)
Управление происходит через IPC socket (--input-ipc-server=/tmp/mpv-socket)
После запуска трека терминал сразу возвращается в готовность
Можно выполнять другие команды параллельно
Преимущества:
✅ Терминал свободен для других задач
✅ Можно управлять воспроизведением из другого окна терминала
✅ Скрипты могут запускать музыку и продолжать работу
Альтернатива (опционально):
Режим --blocking — терминал занят пока играет трек (для скриптов)
Переключатель через флаг: play --blocking "song"
Вопрос 2: Что будет когда композиция закончит играть?
Ответ: Настраиваемое поведение через конфиг.
Режим
Описание
Команда
stop Остановить после трека (по умолчанию) play --on-end stop
next Автовоспроизведение следующего из истории/очереди play --on-end next
loop Зациклить текущий трек play --on-end loop
queue Взять следующий из очереди воспроизведения play --on-end queue

Дополнительно:
Уведомление в терминал: «Трек завершён. Следующий? (y/n)»
Уведомление в десктоп (через notify-send)
Логирование в историю с отметкой о завершении

~/.playmusic/state.json
Хранит: текущий трек, позиция, длительность, громкость, режим on-end.
